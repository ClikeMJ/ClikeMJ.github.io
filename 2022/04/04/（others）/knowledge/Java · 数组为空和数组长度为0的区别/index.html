Java · 数组为空和数组长度为0的区别

Created2022-04-04|Updated2022-04-04|others
|Word count:674|Reading time:2min|Post View:

1.数组为空和长度为0

  • 假设一个方法返回一个数组，如果它返回null，则调用方法必须先判断是否返回null，才能对放回数组进一步处理，而如果返回空数组，则无须null引用检查

  • 鉴于此，返回数组的方法在没有结果时我们通常返回空数组（eg：new一个空数组results），而不是null（不确定的数组nums），这样做对于函数调用者的处理比较方便

  • 关于数组为空和长度为0的总结：

    • 返回值为null并不会有什么问题，但是在“解引用”（dereference）时，调用者没有对null进行判断就会出现NullPointerException。
    • 在返回值为数组或者集合时，尽量返回长度为零的数组或者集合，而不是null，这样在调用时就能简化代码，减少不必要的麻烦，并且不必担心NullPointer异常（除非这对性能会造成很大的影响）。
    • 在编写方法时，如果返回值是数组或者集合，就尽量返回长度为零的数组或者集合，而不是null。而调用方法时，还是要注意对null情况的判断
    • 此外，如果返回值为空对象时没有意义的话，就可以考虑直接抛出异常

2.判断使用顺序

  • 一般先判断 nums 是否为空，然后再判断长度是否为0，否则可能报空指针异常

  • || 是短路或，前面表达式若为True，则后面的就不执行
